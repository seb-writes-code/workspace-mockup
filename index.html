PLT-142

Implement unified search across all content types

In Progress
▲ High
core
Assigned C Chris
Cycle Sprint 14

Users need a single search that works across all content types — issues, docs, threads, code, and PRs. This is one of the key differentiators: instead of searching four different tools, you search once and find everything.

The search should support natural language queries, filters (e.g. type:doc author:chris), and show results grouped by content type with rich previews. See the full spec in ☰ Search Architecture Spec and the design explorations in ☰ Search UI Explorations.

We should index content incrementally as it's created/updated, not batch. Full-text search with pg_trgm gets us started, we can add a dedicated search engine later if needed.
